C
Claude

At its core, causal inference answers one question:

"What happens to Y if I change X?"

This is fundamentally different from asking "What is Y when I observe X?" The first is a question about intervention. The second is about correlation.

The Fundamental Problem

We can never observe what would have happened to the same person under a different treatment. For every unit $i$ and treatment $d$, there exists a potential outcome $Y_i^{(D=d)}$. The causal effect is $\delta_i = Y_i^{(D=1)} - Y_i^{(D=0)}$. But you only ever see one of these.

Why Not Just Compare Groups?

The naive estimator $E[Y|D=1] - E[Y|D=0]$ compares those who happened to get treated vs. not. But these groups may differ systematically. The course's discount example: discounts offered during low-sales weeks made the naive estimate negative (-0.5), even though the true effect was positive (+0.5).

Two Paths Forward

  1. Run an experiment — randomize D so it's independent of everything else
  2. Use observational data carefully — encode causal knowledge as a graph, find the right variables to adjust for

The entire course is about path 2. Can you explain in your own words why the naive estimator can give you the wrong sign?

T
Taewan

In the naive estimator, the difference between the treatment and control groups can be systematic. The assignment mechanism has to be random in order for this naive estimator to equal the true estimator.

C
Claude

Correct When assignment is random, $D$ is independent of potential outcomes — the groups are comparable. The key equation:

$\delta_{\text{naive}} = \delta + \underbrace{E[Y^{(D=0)}|D=1] - E[Y^{(D=0)}|D=0]}_{\text{selection bias}}$

The naive estimator equals the true effect plus the difference in baseline outcomes between groups. Randomization makes that second term zero. Without it, you're measuring the causal effect plus how different the groups already were.

— Causal Graphs —
C
Claude

Now let's move to the graph-based framework. There are exactly three building blocks in every causal graph:

StructureExampleKey Property
ChainX → Z → YConditioning on Z blocks the path
ForkX ← Z → YConditioning on Z blocks the path
ColliderX → Z ← YConditioning on Z opens the path

The first two are intuitive — hold the middle variable fixed, the endpoints become independent. The collider is the trap: blocked by default, but conditioning on it creates spurious association.

T
Taewan

Explain in more detail: "conditioning on." Is it equivalent to "adjusting" and "blocking the path"? Is the goal to block all paths from X to Y except the front door?

C
Claude

"Conditioning on," "adjusting for," and "controlling for" all mean the same thing — holding a variable fixed when analyzing data. Three equivalent ways:

  1. Subsetting: Split data by Z, analyze each subset separately
  2. Including in regression: $\beta_X$ in $Y = \beta_X X + \beta_Z Z$ gives X's effect after accounting for Z
  3. Sample selection: If your dataset only contains admitted students, you are implicitly conditioning on admission — even without including it in a regression

Important: Conditioning is a tool — it can block or open depending on structure. It doesn't always block. For chains and forks, it blocks. For colliders, it opens.

And yes — the goal is to block all non-causal (back-door) paths while keeping causal paths open. The remaining association is the causal effect.

T
Taewan

By only looking at admitted students, we are conditioning on Z=1. Knowing a student is admitted gives us information about the relationship between X and Y: if the student has a low X, then Y is likely to be high.

C
Claude

Exactly In the full population, math and writing are independent. But restricting to admitted students creates an informational constraint: they got in somehow. Low math implies high writing, high math implies low writing. A spurious negative correlation appears.

The notebooks confirmed this numerically:

  • Full population correlation: 0.012 (basically zero)
  • Admitted students only: -0.290 (strongly negative)
— d-Separation & Back-Door Criterion —
C
Claude

d-Separation: A path between X and Y is blocked by set Z if somewhere along it there is:

  1. A chain or fork node that is in Z (you blocked it), OR
  2. A collider node not in Z with no descendant in Z (naturally blocked)

Back-Door Criterion for choosing adjustment set Z:

  1. No descendant of X is in Z
  2. Z blocks all back-door paths (paths entering X from behind)

Let's test your understanding. Consider this graph:

U ──→ X ──→ M ──→ Y │ ↑ └─────────────────┘

U causes both X and Y. X causes Y through M. What's a valid adjustment set for the causal effect of X on Y?

T
Taewan

Adjustment set: none. X → M → Y is a chain structure — conditioning on M will block the causal path from X to Y, closing the front-door path.

C
Claude

Half right Your chain reasoning is correct — conditioning on M would block the causal path. But "none" is wrong.

The back-door path $X \leftarrow U \rightarrow Y$ is a fork that's open by default. U is confounding. The correct answer: adjust for {U}.

Adjustment setBack-door X←U→YCausal X→M→YVerdict
NoneOpen (confounded!)OpenBiased
{U}BlockedOpenCorrect
{M}OpenBlockedBad — kills causal signal
{U, M}BlockedBlockedBad — kills causal signal

Key: "Do nothing" means confounding flows freely through forks. You need to actively block back-door paths. And the goal is not d-separation of X and Y — that would remove all association. You want to block non-causal paths while keeping causal paths open.

— Back-Door Adjustment: Numerical Example —
T
Taewan

Proceed with the next steps. Give me a numerical example.

C
Claude

E-commerce example: Does offering a discount (X) increase purchases (Y)? Loyalty tier (Z) confounds — high-loyalty customers both receive more discounts and buy more.

Z (loyalty)X (discount)CountPurchase rate
LowNo discount4000.20
LowDiscount1000.40
HighNo discount1000.60
HighDiscount4000.80

Naive estimate: 0.44 — inflated, includes the effect of loyalty.

BDA (correct):

$E[Y|do(X=1)] = 0.40 \times 0.5 + 0.80 \times 0.5 = 0.60$
$E[Y|do(X=0)] = 0.20 \times 0.5 + 0.60 \times 0.5 = 0.40$
$\delta_{\text{BDA}} = 0.60 - 0.40 = 0.20$

Within each stratum, the discount effect is 0.20. The naive was 0.44 because the discount group was dominated by high-loyalty customers.

— The Support Assumption —
T
Taewan

Explain the support checking process in detail. What does the propensity score distribution look like? What does distribution overlap mean?

C
Claude

The support assumption (aka positivity):

$0 < P(X=x \mid Z=z) < 1$ for all $z$ where $P(Z=z) > 0$

For every confounder value where people exist, there must be a nonzero probability of each treatment level. If low-loyalty customers never get discounts, $P(X=1|Z=0) = 0$ — the counterfactual is unobserved for the entire stratum.

You check this by plotting propensity score distributions for each group. The overlap region is where both distributions have positive density — that's where causal inference is possible. The danger zones are the tails where only one group exists.

MethodWithout support
StratificationCan't compute — cell is empty
MatchingCan't find a match
IPWWeight = 1/0 = ∞ — blows up
Regression / MLExtrapolates — trusts the model in unseen regions
— Continuous / High-Dimensional Z —
T
Taewan

Let's move on to continuous or high-dimensional Z problems.

C
Claude

Everything so far used discrete, low-dimensional Z. The BDA formula $\sum_z E[Y|X=x, Z=z] \cdot P(Z=z)$ breaks in two ways:

  1. Continuous Z → infinite strata. You can't compute $E[Y|X=1, Z=42{,}387.51]$. Binning creates residual confounding.
  2. High-dim Z → curse of dimensionality. 10 binary confounders = $2^{10} = 1{,}024$ strata. Most cells empty.

Four solutions, each handling Z differently:

MethodStrategy
MatchingFind twins via distance metric or propensity score (1-D)
IPWCollapse Z into propensity score, reweight
Regression/BDALet the model learn the functional form
Double MLML models partial out Z from both sides

Matching and IPW reduce dimensionality (Z → 1 scalar). Regression and Double ML bypass it (let flexible models handle the conditioning).

— Matching Methods —
C
Claude

Exact Matching: Match on identical Z. Problem: curse of dimensionality — most units have no exact match.

Nearest-Neighbor: Use Mahalanobis distance $d(Z_i, Z_j) = \sqrt{(Z_i - Z_j)^\top S^{-1} (Z_i - Z_j)}$. Calipers set a maximum allowed distance — if no match within range, drop the unit rather than make a bad match.

Propensity Score Matching (Rosenbaum & Rubin, 1983): Matching on the single scalar $e(Z) = P(X=1|Z)$ is sufficient. 50 covariates → 1 number.

Can you explain: why is matching on the propensity score sufficient, rather than needing to match on all of Z?

T
Taewan

What does Control (X=0) group's propensity score P(X=1|Z) mean? Density is the number of observations for each propensity score? Most similar people are also treated — does it mean that among the near-one propensity score group, most are in group X=1? Is equivalent density a pre-requisite for propensity score matching? What about score 0.3, where the density for X=0 is much larger than X=1?

C
Claude

P(X=1|Z) for controls: Every unit gets a propensity score — it's the predicted probability of being treated given their covariates. Control units still have a score; it just means that despite, say, a 70% chance, they happened not to be treated.

Density: Relative frequency (smoothed histogram), not raw count. Height shows concentration at that score value.

Near-one = mostly treated: Yes, exactly. At $e(Z) = 0.95$, 95% of similar units are treated. Very few controls exist there — matching becomes hard at the extremes.

Equal density not required. At score 0.3, 200 controls and 20 treated is fine. PSM works by pairing individuals, not averaging groups. Pick a treated unit, find the nearest control, record the difference. Controls can be reused (matching with replacement). The only problem is when one group has zero units — that's a support violation.

T
Taewan

Why is matching on the propensity score sufficient, rather than matching on all of Z? And why can't we just match to a nearest unit when one group has zero at a given propensity score?

C
Claude

The Rosenbaum-Rubin theorem:

If $X \perp\!\!\!\perp (Y^{(0)}, Y^{(1)}) \mid Z$, then $X \perp\!\!\!\perp (Y^{(0)}, Y^{(1)}) \mid e(Z)$

Intuition: The propensity score is a sufficient summary of Z for removing confounding. Among all units with $e(Z) = 0.4$ — regardless of their specific Z values — treatment is effectively a coin flip with 40% probability. Some are young, rich, urban. Others are old, poor, rural. But within this group, who got treated is as good as random.

As for matching when one group is empty: you can, but if the nearest control has $e(Z) = 0.55$ and your treated unit has $e(Z) = 0.95$, these are systematically different people. The "as-good-as-random" argument breaks down. That's why calipers exist — drop the unit rather than make a bad match. But dropping units changes the estimand to ATT for the matchable subpopulation.

— Inverse Propensity Weighting —
C
Claude

Matching finds twins. IPW takes a different approach: reweight the entire sample so it looks like an experiment. Each unit gets weight $w_i = 1/P(X = x_i \mid Z_i)$.

The "surprising units" intuition: A high-loyalty customer who didn't get a discount is unusual and highly informative. IPW upweights these rare units:

Unit typeP(X|Z)WeightWhy
High-loyalty, treated0.901.1Expected — low info
High-loyalty, untreated0.1010.0Surprising — high info
Low-loyalty, untreated0.801.25Expected — low info
Low-loyalty, treated0.205.0Surprising — high info

Can you explain: how does IPW relate to the back-door adjustment formula?

T
Taewan

BDA uses P(Z=z) as a weight and calculates the weighted sum of Y given Z=z (back-door conditioned); a weighted sum across different z values. IPW uses a weight P(X|Z) in the denominator, boosting the lower-chance event outcomes; treating such events as having higher information value; sum across different x value groups (i.e., X=0, 1, etc.) while z is given.

C
Claude

Good intuition on both. One refinement: IPW doesn't condition on Z in the summation. It sums over all units in the treatment group, using the weight to undo confounding without ever stratifying. That's its power — works without binning Z.

They're algebraically equivalent, just rearranged:

  • BDA: $E[Y|do(X=x)] = \sum_z E[Y|X=x, Z=z] \cdot P(Z=z)$ — stratify, weight by population share
  • IPW: $E[Y|do(X=x)] = \frac{1}{N}\sum_{i: x_i=x} \frac{y_i}{P(X=x_i|Z_i)}$ — weight units individually, no strata

Same formula, different failure modes: BDA breaks when Z is high-dimensional (empty strata). IPW breaks when $e(Z)$ is extreme (weights explode).

T
Taewan

IPW sums over all units in the treatment group only? "i" means every unit, and there's N of them? What does $i: x_i = x$ mean in the summation formula? How does IPW take the control group into account (i.e., surprisingly untreated)? How would you write $z_i$ and $P(X=0|Z=z_i)$ for control group + high loyalty?

C
Claude

The ATE formula has two separate sums:

$\hat{\delta}_{\text{IPW}} = \underbrace{\frac{1}{N} \sum_{i:\, x_i=1} \frac{y_i}{e(Z_i)}}_{\text{treated only}} - \underbrace{\frac{1}{N} \sum_{i:\, x_i=0} \frac{y_i}{1 - e(Z_i)}}_{\text{control only}}$

$i: x_i = 1$ means "loop over treated units only" — it's a filter on the summation. The "surprisingly untreated" units appear in the second sum. $N$ = total sample size (both groups).

Control + High Loyalty, concretely:

  • $x_i = 0$ (control), $Z_i = \text{High}$, $e(Z_i) = 0.80$
  • Denominator: $P(X=0|Z=\text{High}) = 1 - 0.80 = 0.20$
  • Weight: $1/0.20 = 5.0$ — 80% of similar people got treated, but this one didn't. Surprising. Represents 5 similar people.

General rule: denominator is always the probability of what actually happened:

$w_i = \begin{cases} 1/e(Z_i) & \text{if treated} \\ 1/(1-e(Z_i)) & \text{if control} \end{cases}$
— Regression for Causal Inference —
C
Claude

Matching and IPW model treatment assignment. Regression flips this: model the outcome directly.

For $Y = \beta_0 + \beta_X X + \beta_Z Z + \epsilon$, the claim is $\beta_X$ = ATE. Plug into BDA: at any fixed $Z = z$, the difference $E[Y|X=1,Z=z] - E[Y|X=0,Z=z] = \beta_X$ — the Z terms cancel identically. OLS is doing BDA implicitly.

Covered: naive regression bias, heterogeneous effects, interaction terms
T
Taewan

How do the Z terms cancel? How can we ensure each corresponding term is the same between X=1 and X=0? What does it mean that the treatment effect $\delta$ varies with Z?

C
Claude

At fixed $Z = z$:

$E[Y|X=1, Z=z] = \beta_0 + \beta_X + \beta_Z z$
$E[Y|X=0, Z=z] = \beta_0 + \beta_Z z$

Subtract: $\beta_0$ cancels. $\beta_Z z$ cancels. What's left is $\beta_X$ — and it's the same for every z. The model assumes Z's contribution to Y is identical regardless of treatment. Only $\beta_X X$ changes when X flips.

"$\delta$ varies with Z": The treatment works differently for different people. Example: a discount boosts purchases by 0.30 for low-loyalty but only 0.10 for high-loyalty. Now $\delta(z) \neq$ constant. A constant-effect OLS gives a weighted average — but weighted by Z's distribution within the treated group, not the population. So $\beta_X \neq$ ATE.

Fix 1: interaction term $\beta_{XZ} X \cdot Z$. Fix 2: BDA predict-and-average — fit any model, predict under $X=1$ and $X=0$ for everyone, average the differences. Handles heterogeneity automatically.

— Model Extrapolation —
C
Claude

Can you explain: why is model-based extrapolation dangerous, and how does it relate to the support assumption?

T
Taewan

Model-based extrapolation is dangerous because regression's prediction solely depends on the existing data, even though the data points of the group that is of interest don't exist, which may fail to capture unique characteristics of the group if it is fundamentally different. This applies the same from the support assumption, where stratification won't work without any data points in a segment to compare the treatment effect.

C
Claude

Strong answer. Two refinements:

1. Sharpen "depends on existing data." All methods depend on data. The specific danger is that regression imposes a functional form and extends it into regions with no data. "Y increases linearly with Z in [0.2, 0.8], so it must continue at Z = 0.95" — but maybe the relationship curves or reverses. Matching/IPW would refuse. Regression fills in the blank with an assumption and gives a confidence interval as if reliable.

2. "Fundamentally different" has a name: lack of common support. The hierarchy of honesty:

— Double ML —
C
Claude

BDA requires correctly specifying $E[Y|X,Z]$. With complex Z you'd use ML, but ML regularization introduces bias that contaminates $\hat{\delta}$ and invalidates confidence intervals.

Double ML residualizes both sides:

  1. Model $\hat{E}[Y|Z]$ and $\hat{E}[X|Z]$ (the propensity score!) with any ML
  2. Compute residuals: $\tilde{Y}_i = Y_i - \hat{E}[Y|Z_i]$, $\tilde{X}_i = X_i - \hat{E}[X|Z_i]$
  3. Regress: $\hat{\delta} = \sum \tilde{X}_i \tilde{Y}_i / \sum \tilde{X}_i^2$

Cross-fitting prevents ML bias from leaking through: train on all data except fold k, predict residuals on held-out fold k.

Covered: partially linear model assumption, Frisch-Waugh-Lovell connection
T
Taewan

The residual regression outcome $\delta$ is equivalent to $\beta_1 = \text{cov}(x, y)/\text{var}(x)$ in a simple linear regression? The slope — one unit change of x's effect on y?

Does Double ML assume a linear model, given that $\delta$ is constant? How can this fit nonlinear models? Is the $g(Z)$ term only subject to ML, while $\delta$ is estimated by linear regression?

What is CATE, how does it relate to R-Learner, and how is it implemented with varying treatment effects across Z?

C
Claude

Yes, exactly. Since residuals are mean-zero, $\sum \tilde{X}_i \tilde{Y}_i / \sum \tilde{X}_i^2 = \text{cov}(\tilde{X}, \tilde{Y}) / \text{var}(\tilde{X})$. One unit of "unexplained-by-Z treatment variation" causes $\delta$ units of "unexplained-by-Z outcome variation."

Partially linear, not fully linear. The model is $Y = \delta X + g(Z) + \epsilon$ — linear in X (one coefficient), but $g(Z)$ can be anything. Division of labor: ML handles complex $g(Z)$ (only needs good prediction); OLS handles $\delta$ (valid inference).

CATE = Conditional Average Treatment Effect: $\tau(z) = E[Y^{(1)} - Y^{(0)} | Z = z]$. Double ML gives one number; CATE gives a function.

R-Learner: Compute residuals as in Double ML, then fit $\hat{\tau}(Z)$ minimizing $\sum_i (\tilde{Y}_i - \tau(Z_i) \cdot \tilde{X}_i)^2$. Double ML asks "what's the single slope?" R-Learner asks "how does the slope change with Z?"

T
Taewan

Explain in detail why "ML models are biased by design" and why this matters for causal inference but not for prediction.

C
Claude

Every model's error: Error = Bias² + Variance + Noise. ML deliberately introduces bias to reduce variance — LASSO shrinks coefficients, tree depth limits prevent memorization, dropout forces redundancy.

Fine for prediction: If adding bias reduces variance by more than it costs, total error decreases. A house price model that's systematically $10K low but stable beats one that's unbiased but swings $200K.

Fatal for causal inference: You need an unbiased $\hat{\delta}$ and valid CIs. LASSO shrinks $\hat{\beta}_X$ toward zero. Your CI might be $[0.08, 0.16]$ when truth is $0.25$ — narrow, precise, doesn't contain the truth. Regularization bias doesn't shrink with more data. The CI is permanently centered on the wrong value.

Double ML's fix: Use ML only for nuisance models (bias is OK — you just need good prediction). Estimate $\delta$ with OLS on residuals (one parameter, no regularization). Cross-fitting ensures the bias doesn't leak.

— Sensitivity Analysis —
C
Claude

Every method assumes the DAG is correct. Sensitivity analysis asks: what if we're wrong?

"How strong would an unmeasured confounder U have to be to explain away our estimated treatment effect?"

If $U \to X$ and $U \to Y$ exist unblocked, the bias is $\beta_U \cdot (E[U|X=1] - E[U|X=0])$. The breakdown point $\alpha^*$ is the minimum confounder strength that zeros out the effect.

BreakdownInterpretationVerdict
Very largeNeed confounder stronger than anything observedRobust
SmallA mild confounder kills the effectFragile
NegativeAdding confounding strengthens the effectVery robust

Analogy: "My bridge holds 10 tons. Sensitivity analysis says it survives up to 25 tons. I don't know if a 25-ton truck will ever cross — but I know my safety margin."

— Putting It All Together —
T
Taewan

Give me a full method comparison — "when to use which" — to tie everything together for the exam.

C
Claude

The Decision Tree

  1. Can you run an experiment? → Randomize. Naive estimator + bootstrap. Done.
  2. Valid back-door set Z? → No: need IV/RDD/diff-in-diff (not on this exam).
  3. What does Z look like? Discrete → Stratification. Continuous → Matching. High-dim → step 4.
  4. Which modeling approach?
    • Trust linear model → OLS
    • Trust outcome model → BDA predict-and-average
    • Trust propensity model → IPW
    • Don't trust either → Doubly-robust
    • Need valid CIs with ML → Double ML
    • Want $\tau(Z)$ → R-Learner
  5. Always do sensitivity analysis.

One-Sentence Summaries

  • Stratification: Split by Z, compare within strata, weight by P(Z)
  • Matching: Find a twin, compare outcomes
  • IPW: Reweight by 1/P(X|Z) to simulate an experiment
  • OLS: Fit a line, read off $\beta_X$
  • BDA: Fit any model, predict under X=1 and X=0, average
  • Double ML: ML handles confounding, OLS estimates $\delta$
  • Doubly-robust: Belt and suspenders — consistent if either model is correct

Three Assumptions Everything Rests On

  1. Graph structure — DAG is correct. No statistical test for this.
  2. No unmeasured confounders — Sensitivity analysis stress-tests this.
  3. Support/overlap — $0 < P(X=1|Z) < 1$. Checkable via PS distributions.
