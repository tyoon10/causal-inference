Course Architecture

The course builds a complete pipeline from defining causal effects to estimating them from data:

1
Define the causal question — Potential outcomes $Y_i^{(D=d)}$, individual effect $\delta_i = Y_i^{(D=1)} - Y_i^{(D=0)}$, ATE $\delta = E[\delta_i]$
2
Encode causal knowledge as a graph — DAGs, Bayesian networks, functional vs probabilistic DGPs
3
Identify the estimand — d-separation, back-door criterion, find adjustment set $Z$
4
Choose an estimator — Stratification, matching, IPW, OLS, ML, Double ML
5
Validate and assess — Support check, balance assessment, sensitivity analysis

Exam Format

Application-focused (no proofs). Three question types:
  • Graph-based: Draw/analyze DAGs, find back-door sets, identify d-separations
  • Computational: Calculate ATE via stratification, matching, IPW, regression
  • Conceptual: Explain when/why methods work, identify violations, compare approaches

Key Assumptions Across All Methods

SUTVA Core
Stable Unit Treatment Value Assumption. No interference between units — one unit's treatment doesn't affect another's outcome. Also: no hidden treatment variations (one version of each treatment level).
Consistency Core
If unit $i$ receives treatment $D=d$, then the observed outcome $Y_i = Y_i^{(D=d)}$. The potential outcome framework is "well-defined" — observed outcomes match the corresponding potential outcomes.
Unconfoundedness / Correct Graph Core
$Y^{(D=1)}, Y^{(D=0)} \perp D \mid Z$ — conditional on the adjustment set, treatment is as-if random. Equivalently: your causal graph is correct, and $Z$ satisfies the back-door criterion.
Positivity / Support Core
$0 < P(D=d \mid Z=z) < 1$ for all $z$ with $P(Z=z)>0$. Every unit has a nonzero chance of receiving each treatment level. Without this, we can't estimate counterfactuals for some strata.

L1: Foundations of Causal Inference

Potential Outcomes Framework Core
For each unit $i$ and treatment level $d$, there exists a potential outcome $Y_i^{(D=d)}$.
$\delta_i = Y_i^{(D=1)} - Y_i^{(D=0)}$

The individual causal effect $\delta_i$ is the difference between what would happen under treatment vs. control for the same unit.

Fundamental Problem Exam
We can never observe both $Y_i^{(D=1)}$ and $Y_i^{(D=0)}$ for the same unit. We always see exactly one potential outcome — the one corresponding to the treatment actually received. The other is the counterfactual.
ATE, ATT, ATC Exam
$\text{ATE} = E[Y^{(D=1)} - Y^{(D=0)}]$
$\text{ATT} = E[Y^{(D=1)} - Y^{(D=0)} \mid D=1]$
$\text{ATC} = E[Y^{(D=1)} - Y^{(D=0)} \mid D=0]$

$\text{ATE} = P(D=1)\cdot\text{ATT} + P(D=0)\cdot\text{ATC}$

Naive Estimator Bias Warning
$\delta_{\text{naive}} = E[Y \mid D=1] - E[Y \mid D=0] \neq \delta$
The naive estimator equals $\delta$ only when $D \perp (Y^{(D=1)}, Y^{(D=0)})$ — i.e., when treatment is randomly assigned. Otherwise it's biased by confounding.
Randomized Experiments Method
Random assignment ensures $D \perp (Y^{(D=0)}, Y^{(D=1)})$, making the naive estimator unbiased. The switching equation: $Y_i = D_i \cdot Y_i^{(D=1)} + (1-D_i) \cdot Y_i^{(D=0)}$.

Randomization check: Verify treatment/control balance using binomial probability bounds.

Bootstrap Method
Standard: Resample $n$ rows with replacement, compute statistic, repeat. Assumes i.i.d.
Poisson: Assign Poisson(1) weight to each row. Equivalent to standard bootstrap for large $n$.
Dependent-samples: Assign Poisson(1) weights at the cluster level (e.g., per user). Correctly accounts for within-cluster correlation. SE can be 2-3x larger than naive.

The do() Operator

$P(Y \mid do(X=x)) \neq P(Y \mid X=x)$

$P(Y \mid do(X=x))$ is the interventional distribution — the distribution of $Y$ if we physically set $X=x$ for everyone. Graphically: delete all edges into $X$ and set $X=x$.

$P(Y \mid X=x)$ is the observational conditional — the distribution of $Y$ among units that happen to have $X=x$. This includes confounding.

L2: Data Generating Processes & Causal Graphs

Functional vs Probabilistic DGPs Core
Probabilistic: Draw from conditional distributions: $X_1 \sim P(X_1)$, $X_2 \sim P(X_2|X_1)$, ...
Functional (Structural Equations): $X_i = f(\text{Parents}(X_i), \epsilon_i)$ where $\epsilon_i$ are independent noise terms.

Both are statistically equivalent — they produce samples from the same joint distribution. The functional approach makes interventions easier to reason about (just replace an equation).

Bayesian Networks Core
A DAG + conditional distributions. The joint factorizes as:
$P(X_1, ..., X_n) = \prod_i P(X_i \mid \text{Parents}(X_i))$
This factorization implies specific conditional independence relationships that can be read off the graph.
Observational Equivalence Warning
Multiple different causal graphs can produce the same observational distribution. $X \to Y$ and $Y \to X$ are observationally equivalent with just two variables. You cannot determine causal direction from data alone without additional assumptions.
Confounding Example: CTR Exam
Scenario: Title length $X$ appears to affect click rate $Y$, but both are driven by genre preference $Q$ (horror fans prefer 13-word titles and get more clicks).
Graph: $Q \to X$, $Q \to Y$ (no edge $X \to Y$).
Naive: $E[Y|X=x]$ shows a peak at 13 words. Stratified: $E[Y|X=x, Q=q]$ shows NO relationship — the correlation was entirely spurious.

Three Building Blocks of Causal Graphs

Chain: $X \to Z \to Y$ Core
Unconditional: $X \not\perp Y$ (correlated via the path).
Conditioning on Z: $X \perp Y \mid Z$ (path is BLOCKED).

Light switch analogy: If you know the current is flowing ($Z$), knowing the switch position ($X$) tells you nothing new about the light ($Y$).

Caution: If $Z$ is a mediator of the causal effect of $X$ on $Y$, conditioning on $Z$ will make $X$ appear to have no effect — even though it does!

Fork: $X \leftarrow Z \to Y$ Core
Unconditional: $X \not\perp Y$ (spurious correlation via common cause).
Conditioning on Z: $X \perp Y \mid Z$ (path is BLOCKED).

Lemonade & crime: Temperature ($Z$) causes both lemonade sales ($X$) and crime ($Y$). They're correlated but neither causes the other. Conditioning on temperature removes the spurious correlation.

Collider: $X \to Z \leftarrow Y$ Warning
Unconditional: $X \perp Y$ (path is BLOCKED at collider).
Conditioning on Z: $X \not\perp Y \mid Z$ (path is OPENED!).

University admission: Math ($X$) and writing ($Y$) skills are independent. But among admitted students ($Z=1$), they become negatively correlated — if someone was admitted despite bad math, they must be good at writing.

Key rule: Conditioning on a collider (or its descendants) OPENS a previously blocked path.

Summary Table

StructureUnconditionalConditioning on middle node
Chain: $X \to Z \to Y$$X \not\perp Y$$X \perp Y \mid Z$ (BLOCKS)
Fork: $X \leftarrow Z \to Y$$X \not\perp Y$$X \perp Y \mid Z$ (BLOCKS)

L3: d-Separation & Back-Door Criterion

d-Separation Definition Core
A path between $X$ and $Y$ is blocked by a set $Z$ if, along the path, there is either:
  1. A chain or fork node that is IN $Z$ (blocking), OR
  2. A collider node that is NOT in $Z$ and has no descendant in $Z$
$X$ and $Y$ are d-separated by $Z$ if ALL paths between them are blocked.
$X \perp_d Y \mid Z \implies X \perp Y \mid Z$
d-separation in the graph implies conditional independence in the distribution (under faithfulness).
Back-Door Criterion Exam
A set $Z$ satisfies the back-door criterion for the effect of $X$ on $Y$ if:
  1. No descendant of $X$ is in $Z$
  2. $Z$ blocks all back-door paths from $X$ to $Y$ (paths that start with an arrow INTO $X$)

A "back-door path" is any path from $X$ to $Y$ that has an arrow pointing into $X$ (i.e., $\leftarrow X$). These are non-causal paths that create confounding.

Back-Door Adjustment Formula Exam
If $Z$ satisfies the back-door criterion:
$P(Y \mid do(X=x)) = \sum_z P(Y \mid X=x, Z=z) \, P(Z=z)$
In expectation form:
$E[Y \mid do(X=x)] = \sum_z E[Y \mid X=x, Z=z] \, P(Z=z)$

This converts the interventional distribution into observational quantities we can estimate from data.

Finding Back-Door Sets: Step-by-Step

1
Identify all back-door paths from $X$ to $Y$: paths with an arrow INTO $X$ at the start
2
Check for colliders on each path. A collider blocks the path by default; don't include it (or its descendants) in $Z$
3
Find non-descendant nodes that block all remaining open back-door paths
4
Verify: No node in $Z$ is a descendant of $X$, and all back-door paths are blocked

Stratification & Coarsening

Stratification = Direct BDA Implementation

For each stratum $z$, compute $E[Y|X=x, Z=z]$, then weight by $P(Z=z)$.

$\hat{\delta} = \sum_z \left(E_N[Y \mid X=1, Z=z] - E_N[Y \mid X=0, Z=z]\right) P_N(Z=z)$

Coarsening trade-off: Fine strata = less bias, more variance (sparse data per cell). Coarse strata = more bias (residual confounding), less variance (more data per cell).

Voter outreach example: County-level stratification (3000+ strata) was too fine with $N=2000$. State-level (50 strata) gave good estimates despite some residual confounding.

L4: Conditioning-Based Methods

Exact Matching Method
For each unit, find a match with the same confounder values but opposite treatment.
No model assumptions Curse of dimensionality Requires discretization

Problem: With 5 confounders and 5 bins each: $5^5 = 3125$ strata. Most will be empty with moderate $N$.

App example: Naive estimate was $-0.04$ (wrong sign!). Exact matching with coarse bins recovered $+0.06$ (true = $0.10$). Residual bias from coarsening.

Nearest-Neighbor Matching Method
Match each unit to its closest counterpart in the opposite treatment group using a distance metric.
$d_{\text{Mahalanobis}}(z_i, z_j) = \sqrt{(z_i - z_j)^T \Sigma^{-1} (z_i - z_j)}$
Works with continuous Z Curse of dimensionality in high-D

Caliper: Reject matches where distance exceeds a threshold. Trades off coverage for match quality.

Propensity Score Matching Method
The propensity score $e(Z) = P(X=1|Z)$ is a balance score: it's sufficient for removing confounding.
$X \perp Z \mid e(Z)$
Match on the 1D propensity score instead of the full $Z$ vector. Avoids curse of dimensionality.
Reduces to 1D matching Requires correct propensity model

Balance assessment: After matching, check that the distribution of $Z$ is similar between treated and matched controls. Use standardized mean difference statistic.

Support Assumption Warning
$0 < P(X=x \mid Z=z) < 1$ for all $z$ with $P(Z=z) > 0$
Without overlap, we can't find matches. Check support visually: plot propensity score distributions for treated vs. control. If they don't overlap somewhere, matching can't work there.

Checking support: (1) Estimate propensity score $e(Z) = P(X{=}1|Z)$ via logistic regression. (2) Plot overlapping histograms for $X{=}1$ vs $X{=}0$. (3) Flag scores below 0.05 or above 0.95 — these create instability.

Danger of Discretization Warning
Coarsely discretizing a continuous confounder only partially blocks the back-door path. Residual variation within bins maintains the association.

Example: In the fork $X_1 \leftarrow X_2 \to X_3$, conditioning on continuous $X_2$ gives $\beta_{X_1}=0$ (correct). Discretizing $X_2$ into 5 bins gives $\beta_{X_1}=0.087$ (biased!).

Bias Amplification Warning
Adjusting for the wrong variables can make bias worse, not better. If a variable is an instrument (causes $X$ but not $Y$ directly), conditioning on it amplifies confounding from unmeasured variables.

Voter example: County contact rate was NOT a confounder (it affected $X$ but not $Y$ directly). Stratifying by it amplified bias: estimate went from 0.068 to 0.072 (true = 0.05).

Rule: Only condition on variables that satisfy the back-door criterion. Blindly "controlling for everything" is dangerous.

Matching Estimator Formulas

After matching, construct potential outcomes for each unit:

$\hat{Y}_i^{(D=1)} = \begin{cases} Y_i & \text{if } D_i = 1 \\ Y_{\text{match}(i)} & \text{if } D_i = 0 \end{cases}$
$\hat{\delta}_{\text{ATE}} = \frac{1}{N}\sum_i \left(\hat{Y}_i^{(D=1)} - \hat{Y}_i^{(D=0)}\right)$

For ATT: only match the treated units. For ATC: only match the control units.

$\text{ATE} = P(D=1) \cdot \text{ATT} + P(D=0) \cdot \text{ATC}$

Key Observations

Unequal group sizes are fine

At $e(Z)=0.3$, there may be 10 controls but only 2 treated. That's OK — we match from treated to control, not group-average. Controls can be reused (with replacement).

What $e(Z)$ means for controls

A control unit with $e(Z) = 0.8$ had an 80% predicted chance of being treated — but wasn't. They're rare and valuable: they look like treated units but didn't get treatment.

Density ≠ count (but related)

Density is a smoothed histogram — it shows concentration of units, not raw count. A tall peak at 0.7 means many units cluster there. The area under the curve integrates to 1.

Why near-1 scores are mostly treated

$e(Z)=0.95$ means 95% chance of treatment given covariates. So most units there are treated. Few controls exist — matching gets hard. This is where support problems live.

Why Matching on e(Z) Is Sufficient

The Rosenbaum-Rubin theorem says: if conditioning on all of Z removes confounding, then conditioning on the single scalar $e(Z) = P(X{=}1|Z)$ does too.

$$X \perp\!\!\!\perp (Y^{(0)}, Y^{(1)}) \mid Z \;\;\Longrightarrow\;\; X \perp\!\!\!\perp (Y^{(0)}, Y^{(1)}) \mid e(Z)$$

Click on any propensity score band to see the units inside it. Despite having very different covariate profiles, they share the same treatment probability — so within each band, treatment is as good as random.

Click a colored band to see the units inside that propensity score stratum.

The Compression Intuition

$$Z = (\text{age, income, education, region, ...}) \;\xrightarrow{\;e(\cdot)\;}\; e(Z) = 0.4$$

Many different covariate profiles map to the same propensity score. A young, high-income, urban person and an old, low-income, rural person can both have $e(Z) = 0.4$. What matters is not why they have that probability, but that they do. Within this group, treatment assignment carries no information about Z — confounding is gone.

L5: Weighting, Regression & ML Methods

IPW Estimator — The Full Formula Method

The ATE is computed as two separate sums, one per group, then subtracted:

$\hat{\delta}_{\text{IPW}} = \underbrace{\frac{1}{N} \sum_{i:\, x_i=1} \frac{y_i}{e(Z_i)}}_{\text{estimate of } E[Y|do(X=1)]} \;-\; \underbrace{\frac{1}{N} \sum_{i:\, x_i=0} \frac{y_i}{1 - e(Z_i)}}_{\text{estimate of } E[Y|do(X=0)]}$

Reading the notation

SymbolMeaning
$i: x_i = 1$Sum only over treated units (a filter on the loop)
$i: x_i = 0$Sum only over control units
$N$Total sample size (both groups), NOT group size
$e(Z_i)$$P(X{=}1 \mid Z_i)$ — propensity score of unit $i$
$1 - e(Z_i)$$P(X{=}0 \mid Z_i)$ — probability of being untreated

The general weight rule

The denominator is always the probability of what actually happened:

$w_i = \frac{1}{P(X = x_i \mid Z_i)} = \begin{cases} 1/e(Z_i) & \text{if treated } (x_i = 1) \\ 1/(1 - e(Z_i)) & \text{if control } (x_i = 0) \end{cases}$

Units are "surprising" when their actual treatment status was unlikely given their covariates — the denominator is small, so the weight is large.

Concrete example: Control + High Loyalty

PropertyValue
Actual status$x_i = 0$ (control — not treated)
Covariates$Z_i = \text{High Loyalty}$
Propensity$e(Z_i) = P(X{=}1 \mid Z{=}\text{High}) = 0.80$
Weight denom.$P(X{=}0 \mid Z{=}\text{High}) = 1 - 0.80 = 0.20$

Interpretation80% of similar people got treated, but this unit didn't — surprising, so highly informative. Represents 5 similar people.

Works for any function: Replace $y_i$ with $f(y_i)$ to estimate $E[f(Y) \mid do(X=x)]$ — not just means, but variances, quantiles, etc.

No outcome model needed No stratification needed — works with continuous Z Weights explode when e(Z) near 0 or 1 Correct propensity score model

OLS Regression — Why βX = ATE Method

For linear model $Y = \beta_0 + \beta_X X + \beta_Z Z + \epsilon$, claim: $\beta_X$ is the ATE.

Step-by-step: connecting regression to BDA

Write out the conditional expectation for each treatment value at a fixed $Z = z$:

$E[Y|X{=}1, Z{=}z] = \beta_0 + \beta_X(1) + \beta_Z z$

$E[Y|X{=}0, Z{=}z] = \beta_0 + \beta_X(0) + \beta_Z z$

Subtract (the effect of treatment at this specific $z$):

$E[Y|X{=}1, Z{=}z] - E[Y|X{=}0, Z{=}z] = \beta_X$

$\beta_0$ cancels. $\beta_Z z$ cancels. What's left is $\beta_X$ — and it's the same for every value of $z$. The linear model assumes the treatment effect doesn't depend on Z.

Plug into BDA: $\sum_z \beta_X \cdot P(Z{=}z) = \beta_X \cdot 1 = \beta_X$  ✓

The naive regression trap (omitting Z)

Regress Y on X without Z: $Y = \alpha_0 + \alpha_X X + \epsilon$

$\alpha_X = \underbrace{\beta_X}_{\text{true effect}} + \underbrace{\beta_Z \cdot (E[Z|X{=}1] - E[Z|X{=}0])}_{\text{selection bias in regression form}}$

Bias = how much Z affects Y ($\beta_Z$) × how different treated/control groups are on Z.

Simple, well-understood Assumes constant treatment effect Correct functional form (linearity)
Heterogeneous Treatment Effects Exam

"$\delta$ varies with Z" means the treatment works differently for different people:

Z (loyalty)Effect of discount
Low loyalty$\delta = +0.30$ (big boost — they needed the nudge)
High loyalty$\delta = +0.10$ (small boost — would have bought anyway)

The subtraction doesn't cancel cleanly: $E[Y|X{=}1, Z{=}z] - E[Y|X{=}0, Z{=}z] = \delta(z) \neq \text{constant}$

Constant-effect OLS gives a weighted average of effects — but weighted by Z's distribution within the treated group, not the population. So $\beta_X \neq$ ATE.

Two fixes

Fix 1 — Interaction term: $Y = \beta_0 + \beta_X X + \beta_Z Z + \beta_{XZ} X \cdot Z + \epsilon$. Then $\delta(z) = \beta_X + \beta_{XZ} z$.

Fix 2 — BDA predict-and-average: handles any functional form automatically.

Advanced Regression Methods

Model Extrapolation & Lack of Common Support Warning

The danger: imposing a functional form

All methods use data. The specific danger with regression is that it imposes a functional form (linear, polynomial, etc.) and extends it into regions with no data. Example:

"Y increases linearly with Z in [0.2, 0.8] where I have data, so it must continue linearly at Z = 0.95."
But maybe the relationship curves, saturates, or reverses there. There's no data to check.

Lack of common support

Where $P(X{=}1|Z) \approx 0$ or $\approx 1$, one group has no units. We have no empirical basis for the counterfactual. Regression manufactures one from assumptions; matching and IPW refuse.

Hierarchy of honesty

MethodNo-overlap regionFailure mode
Exact matchingDrops the unitHonest — no estimate
PSM with calipersDrops the unitHonest — no estimate
IPWWeight → ∞Breaks loudly
Regression / BDAExtrapolatesBreaks silently

The silent failure is uniquely dangerous — you get a confident estimate and a confidence interval, with no indication that the answer rests entirely on functional form assumptions rather than data.

IPW-Weighted Causal Regression Method
Train any model on $X$ alone (no $Z$!) with IPW-weighted loss:
$L_{\text{causal}} \propto \sum_i \frac{(y_i - \hat{f}(x_i))^2}{P(X=x_i|Z=z_i)}$

The model learns $E[Y \mid do(X)]$ directly.

Doubly-robust: Include $Z$ AND use IPW weights. Consistent if EITHER model is correct.

Why ML Models Break Causal Inference

ML models are biased by design. Regularization deliberately introduces systematic error to reduce variance. This is fine for prediction — but fatal for causal inference. Toggle to see why.

Double ML: Residualization

Double ML — The Partially Linear Model Method
$Y = \delta X + g(Z) + \epsilon$

Division of labor

PartWhatWho handles itRequirement
$\delta X$Treatment effect (constant)OLS on residualsValid inference, CIs
$g(Z)$Confounding (can be complex)ML (any model)Only needs good prediction

The model is "partially linear" — linear in X (one coefficient $\delta$), but $g(Z)$ can be arbitrarily nonlinear. ML handles the hard part; OLS handles the easy part.

The residualization procedure

  1. Model $\hat{E}[Y|Z]$ — predict Y from confounders alone (any ML model)
  2. Model $\hat{E}[X|Z]$ — predict treatment from confounders (this is the propensity score!)
  3. Compute residuals: $\tilde{Y}_i = Y_i - \hat{E}[Y|Z_i]$ and $\tilde{X}_i = X_i - \hat{E}[X|Z_i]$
  4. Regress: $\hat{\delta} = \sum_i \tilde{X}_i \tilde{Y}_i \;/\; \sum_i \tilde{X}_i^2$

Why the residual formula = OLS slope

The formula $\hat{\delta} = \sum \tilde{X}_i \tilde{Y}_i / \sum \tilde{X}_i^2$ is the OLS coefficient for regressing $\tilde{Y}$ on $\tilde{X}$ (no intercept). Since residuals are mean-zero by construction, this equals $\text{cov}(\tilde{X}, \tilde{Y}) / \text{var}(\tilde{X})$ — the slope. One unit of "unexplained-by-Z treatment variation" causes $\delta$ units of "unexplained-by-Z outcome variation."

Cross-fitting: why it's needed

ML regularization introduces bias. If you predict residuals on the same data you trained on, that bias leaks into $\hat{\delta}$. Cross-fitting (like cross-validation) trains on K-1 folds, predicts on the held-out fold. Held-out predictions are unbiased.

ML handles complex g(Z) Valid CIs via cross-fitting Assumes constant treatment effect
CATE & R-Learner — Heterogeneous Effects Exam

CATE: Conditional Average Treatment Effect

$\tau(z) = E[Y^{(1)} - Y^{(0)} \mid Z = z]$

"The treatment effect for people with covariates $Z = z$." Double ML gives one number ($\delta$) for everyone. CATE gives a function — the effect varies with Z.

R-Learner procedure

  1. Compute residuals exactly as Double ML: $\tilde{Y}_i$, $\tilde{X}_i$
  2. Fit a model $\hat{\tau}(Z)$ that minimizes:
$\sum_i \left(\tilde{Y}_i - \tau(Z_i) \cdot \tilde{X}_i\right)^2$

Intuition: Double ML asks "what's the single slope between $\tilde{X}$ and $\tilde{Y}$?" R-learner asks "what's the slope, and how does it change with Z?"

The model $\hat{\tau}(Z)$ can be a random forest, neural net, etc. — it learns which subgroups benefit most from treatment.

L6: Practical Limitations

Graph Sufficiency Exam
"Is your graph good enough?" The central question in applied causal inference. All methods assume the graph is correct. If you're missing a confounder or have the wrong structure, estimates will be biased.

There is no statistical test for whether your graph is correct. Domain knowledge is essential.

Causal Discovery (PC Algorithm) Method
Learns graph structure from data using conditional independence tests:
  1. Start with a fully connected graph
  2. Remove edges where conditional independence is found
  3. Orient edges using collider detection and propagation rules

Independence tests:

TestData TypeLimitation
Fisher-ZContinuous (+ binary)Assumes linear associations
Chi-squareDiscreteRequires discretizing continuous vars
KCIAny (nonparametric)$O(n^3)$, impractical at scale

Key limitation: Discretizing continuous mediators causes information loss and spurious edges.

Sensitivity Analysis: Stress-Testing Your Estimate

The Core Question Exam

Every method assumes the DAG is correct — that all confounders are measured and adjusted for. Sensitivity analysis asks: what if we're wrong?

"How strong would an unmeasured confounder $U$ have to be to explain away our estimated treatment effect?"

Suppose you've estimated $\hat{\delta} = 0.20$ after adjusting for all observed $Z$. But an unmeasured $U$ exists with $U \to X$ and $U \to Y$, creating an unblocked back-door path. The bias follows the same structure as the naive estimator:

$\text{Bias} = \beta_U \cdot \left(E[U \mid X{=}1] - E[U \mid X{=}0]\right)$

How much $U$ affects $Y$ ($\beta_U$) × how different the groups are on $U$ (imbalance). Your true effect is $\delta_{\text{true}} = \hat{\delta} - \text{Bias}$.

What sensitivity analysis is NOT

  • Does not tell you whether an unmeasured confounder exists
  • Does not fix the bias
  • Quantifies your vulnerability — how much hidden confounding your conclusion can tolerate

Analogy: "My bridge holds 10 tons. Sensitivity analysis says it survives up to 25 tons. I don't know if a 25-ton truck will ever cross — but I know my safety margin."

The Breakdown Point Exam

The key output: the minimum confounder strength that would zero out the effect.

Set $\hat{\delta} - \alpha = 0$ and solve. The breakdown value $\alpha^*$ is the confounding magnitude needed to make the estimated ATE vanish.

The shifted-outcome procedure (from the course)

  1. Start with your estimated ATE (e.g., $\hat{\delta} = 0.20$)
  2. Simulate an unmeasured confounder by shifting outcomes: $Y^{(\alpha)} = Y - \alpha \cdot (2D - 1) \cdot P(D = 1{-}d \mid Z)$
  3. Re-estimate the treatment effect with the shifted $Y^{(\alpha)}$
  4. Increase $\alpha$ until $\hat{\delta}^{(\alpha)} = 0$
  5. That $\alpha^*$ is the breakdown point

Interpreting the breakdown point

Breakdown valueInterpretationVerdict
Very largeNeed a confounder stronger than anything observedRobust
SmallA mild unmeasured confounder would kill the effectFragile
NegativeAdding confounding strengthens the effectVery robust

Connection to the hierarchy of assumptions

All causal methods rest on: (1) Graph structure is correct, (2) No unmeasured confounders, (3) Support/overlap. Sensitivity analysis stress-tests assumption #2. There's no statistical test for it — only this "how bad would it have to be?" reasoning.

When to Use Which Method

1
Can you run an experiment?
→ Yes: Randomize. Naive estimator + bootstrap CIs. No DAG needed. Done.
2
Do you have a valid back-door set $Z$?
→ No: Need IV, RDD, or diff-in-diff (not on this exam). Stop.
3
What does $Z$ look like?
→ Discrete, low-dim (≤ 3 vars): Stratification (direct BDA sum)
→ Continuous, low-dim: Matching (NN/Mahalanobis or PSM). Check overlap first.
→ High-dim or mixed: go to step 4.
4
Which modeling approach?
→ Trust linear $E[Y|X,Z]$, constant effect: OLS
→ Don't trust linearity, trust your outcome model: BDA predict-and-average
→ Don't trust outcome model, trust propensity model: IPW
→ Don't trust either alone: Doubly-robust (IPW + regression)
→ Complex $g(Z)$, need valid CIs: Double ML
→ Want treatment effect heterogeneity $\tau(Z)$: R-Learner
5
Always do sensitivity analysis.
No matter which method: your result rests on the assumption that $Z$ is sufficient. Sensitivity analysis quantifies how much that assumption can be violated before your conclusion breaks.

One-Sentence Summary of Each Method

Stratification — Split the data by $Z$, compare within strata, weight by $P(Z)$
Matching — Find a twin with similar $Z$ (or propensity score), compare outcomes
IPW — Reweight every unit by $1/P(X|Z)$ to simulate an experiment
OLS — Fit a line with $Z$ included, read off $\beta_X$ as the ATE
BDA predict-and-average — Fit any model, predict under $X{=}1$ and $X{=}0$ for everyone, average
Double ML — Let ML handle confounding ($g(Z)$), OLS estimates the causal parameter ($\delta$)
Doubly-robust — Belt and suspenders: consistent if EITHER model is correct
Sensitivity Analysis — Stress-test: how strong must a hidden confounder be to kill the effect?

The Three Assumptions Everything Rests On

1. Graph Structure — The DAG is correct: no missing arrows, no wrong directions. There is no statistical test for this. Domain knowledge is essential. PC algorithm can help discover structure but has limitations (discretization artifacts, can't distinguish some orientations).
2. No Unmeasured Confounders — All back-door paths are blockable with observed $Z$. Untestable from data alone. Sensitivity analysis stress-tests this: "how strong would a hidden confounder need to be?"
3. Support / Overlap — $0 < P(X{=}1|Z{=}z) < 1$ for all $z$ where $P(Z{=}z) > 0$. Both treatment groups must exist at every confounder level. Checkable via propensity score distributions. Without overlap: matching drops units, IPW weights explode, regression extrapolates silently.

Practice Problems

Q1: Given the graph $Z_1 \to X$, $Z_1 \to Y$, $Z_2 \to X$, $Z_2 \to Y$, $X \to Y$, what is a valid back-door set for the effect of $X$ on $Y$?

Back-door paths from $X$ to $Y$: (1) $X \leftarrow Z_1 \to Y$ and (2) $X \leftarrow Z_2 \to Y$.

Valid back-door sets: $\{Z_1, Z_2\}$, $\{Z_1\}$ alone blocks path 1 but not path 2, so we need both. Actually, $\{Z_1\}$ only blocks path 1. $\{Z_2\}$ only blocks path 2. We need $\{Z_1, Z_2\}$.

Check: Neither $Z_1$ nor $Z_2$ is a descendant of $X$. Both back-door paths are blocked.

Q2: In the graph $X \to M \to Y$ with $U \to X$ and $U \to Y$ (U unobserved), is $\{M\}$ a valid back-door set?

No! $M$ is a descendant of $X$, violating condition 1 of the back-door criterion. Additionally, conditioning on $M$ would block the causal path $X \to M \to Y$, which we don't want.

The valid back-door set would be $\{U\}$, but $U$ is unobserved. Without observing $U$, we cannot use back-door adjustment here.

Q3: You observe the following data with binary $X$ and confounder $Z \in \{0,1\}$:
$E[Y|X{=}1, Z{=}0] = 5$, $E[Y|X{=}0, Z{=}0] = 3$, $P(Z{=}0) = 0.6$
$E[Y|X{=}1, Z{=}1] = 8$, $E[Y|X{=}0, Z{=}1] = 7$, $P(Z{=}1) = 0.4$
What is the ATE using the back-door adjustment formula?

$E[Y|do(X{=}1)] = E[Y|X{=}1,Z{=}0] \cdot P(Z{=}0) + E[Y|X{=}1,Z{=}1] \cdot P(Z{=}1)$

$= 5 \times 0.6 + 8 \times 0.4 = 3.0 + 3.2 = 6.2$

$E[Y|do(X{=}0)] = 3 \times 0.6 + 7 \times 0.4 = 1.8 + 2.8 = 4.6$

ATE $= 6.2 - 4.6 = 1.6$

Stratum-level effects: $\delta_{Z=0} = 5 - 3 = 2$, $\delta_{Z=1} = 8 - 7 = 1$.

$\text{ATE} = 2 \times 0.6 + 1 \times 0.4 = 1.6$. Same answer.

Q4: Why does conditioning on a collider introduce bias? Give an intuitive explanation.

A collider $X \to Z \leftarrow Y$ blocks the path between $X$ and $Y$ by default. But conditioning on $Z$ (or selecting on $Z$) opens the path.

Intuition: Knowing the collider's value constrains the relationship between its parents. If $Z = X + Y$ and you know $Z = 10$, then knowing $X = 8$ tells you $Y = 2$. The parents become dependent given the collider's value, even though they were independent unconditionally.

Real-world: Among admitted students (collider = admission), math and writing skills appear negatively correlated, even though they're independent in the general population.

Q5: You have a propensity score $P(X{=}1|Z{=}z_i) = 0.8$ for unit $i$ with $X_i = 1$. What is the IPW weight for this unit? What about a unit with $P(X{=}1|Z) = 0.8$ and $X = 0$?

For unit with $X=1$: $w_i = 1/P(X{=}1|Z) = 1/0.8 = 1.25$

For unit with $X=0$: $w_i = 1/P(X{=}0|Z) = 1/(1-0.8) = 1/0.2 = 5.0$

The control unit at $Z$ where treatment is likely gets a much higher weight (5x vs 1.25x). This makes sense: seeing a control unit where almost everyone is treated is very informative about the counterfactual.

Q6: Explain why $\beta_X$ in OLS ($Y = \beta_X X + \beta_Z Z + \epsilon$) equals the ATE when the treatment effect is constant, but not when it varies with $Z$.

Constant effect: The BDA formula gives $E[Y|do(X{=}x)] = \sum_z E[Y|X{=}x, Z{=}z]P(Z{=}z)$. Plugging in the linear model: $E[Y|X{=}1,Z{=}z] - E[Y|X{=}0,Z{=}z] = \beta_X$ for all $z$. So $\delta = \sum_z \beta_X P(Z{=}z) = \beta_X$.

Heterogeneous effect: If the true model is $Y = \delta(Z) \cdot X + g(Z) + \epsilon$, fitting a constant-effect model forces $\delta(Z) \to \beta_X$. OLS minimizes MSE and produces a variance-weighted average of $\delta(Z)$, which doesn't equal $E[\delta(Z)]$ in general. The weighting depends on the propensity score distribution, biasing toward values of $Z$ where treatment assignment is most variable.

Fix: Add interaction term ($\beta_{XZ} XZ$) or use BDA predict-and-average.

Q7: What are the three steps of Double ML, and why is cross-fitting necessary?
  1. Cross-fit: Split data into $K$ folds
  2. Residualize: On each held-out fold, compute $\tilde{Y}_i = Y_i - \hat{E}[Y|Z_i]$ and $\tilde{X}_i = X_i - \hat{E}[X|Z_i]$ using models trained on other folds
  3. Estimate: Regress $\tilde{Y}$ on $\tilde{X}$: $\hat{\delta} = \sum \tilde{X}_i \tilde{Y}_i / \sum \tilde{X}_i^2$

Why cross-fitting? ML models use regularization (early stopping, depth limits, L2 penalties), which introduces bias in predictions. If nuisance models predict on their own training data, this bias leaks into the causal estimate. Cross-fitting ensures nuisance models never predict on data they were trained on, preventing overfitting bias from contaminating $\hat{\delta}$.

Q8: In sensitivity analysis, the breakdown point $\alpha^*$ for an estimated ATE of 1.0 is $\alpha^* \approx 1.0$. What does this mean practically?

The breakdown point is the minimum violation of unconfoundedness needed to explain away the entire treatment effect (reduce ATE to zero).

$\alpha^* \approx 1.0$ means: an unmeasured confounder would need to create a 1.0 unit difference in expected potential outcomes between treated and control groups (within each stratum of observed confounders) to fully explain the observed effect.

Whether this is a lot or a little depends on domain knowledge. If $Y$ is measured in standard deviations, a 1.0 unit violation is quite large (the unmeasured confounder would need to shift expected outcomes by a full standard deviation). If $Y$ is in dollars, 1.0 might be trivially small.

Key insight: Sensitivity analysis doesn't tell you whether unconfoundedness holds. It tells you how robust your conclusion is to violations.

Q9: You want to estimate the causal effect of a study app on student pass rates. Your confounders are: household income (continuous), parent education (5 categories), GPA (continuous), commute time (continuous), hobbies (continuous). Which estimation method would you recommend and why?

Not exact matching: $5^5 = 3125$ strata (with 5 bins each) — too many for most sample sizes. The curse of dimensionality.

Options:

  • Propensity score matching: Reduces 5D matching to 1D. Fit logistic regression for $P(\text{app}|Z)$, match on the score. Check balance post-matching.
  • OLS with all confounders: If linear model is reasonable. $\beta_{\text{app}}$ = ATE if constant effect.
  • IPW: Weight by inverse propensity. Good if you trust the propensity model but not an outcome model.
  • Double ML: Best if relationships are nonlinear. Uses flexible ML for nuisance models while giving valid CIs for the causal effect.

Always: Check support (propensity score overlap between users and non-users), and run sensitivity analysis afterwards.
